# norm4j Skill Demo

This document demonstrates how to use the norm4j skill to work with the norm4j framework.

## Quick Start

### 1. Create a Basic Entity

```bash
# Create a Customer entity
norm4j:create-entity --name Customer --package com.example.entities --tableName customers
```

This generates a complete entity class with proper annotations:

```java
package com.example.entities;

import org.norm4j.Table;
import org.norm4j.Id;
import org.norm4j.GeneratedValue;
import org.norm4j.GenerationType;
import org.norm4j.Column;

@Table(name = "customers")
public class Customer {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private int id;

    @Column(nullable = false)
    private String name;

    // Getters and Setters
    public int getId() {
        return id;
    }

    public void setId(int id) {
        this.id = id;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
}
```

### 2. Generate DDL

```bash
# Generate PostgreSQL DDL
norm4j:generate-ddl --package com.example.entities --dialect postgresql
```

This generates the SQL to create the table:

```sql
-- Generated DDL for com.example.entities (postgresql)
-- This would be generated by scanning the package for @Table classes
-- and analyzing their annotations

CREATE TABLE customers (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_customers_name ON customers(name);
```

### 3. Create Initial Migration

```bash
# Create initial migration
norm4j:create-migration --version v0.1 --description "Initial customer schema"
```

This generates migration code:

```java
// Migration v0.1
// Initial customer schema

new SchemaSynchronizer(tableManager)
    .version("v0.1")
        .description("Initial customer schema")
        .schema()
            .enableAutoCreation(true)
        .endSchema()
        // Add your migration SQL here
        .execute("ALTER TABLE customer ADD COLUMN email VARCHAR(255)")
    .endVersion()
    .apply();
```

### 4. Generate DTO and Mapper

```bash
# Generate DTO
norm4j:generate-dto --entity Customer --package com.example.dtos --fields id,name,email

# Generate mapper
norm4j:generate-mapper --entity Customer --dto CustomerDTO --mappings id:id,name:name,email:email
```

This generates:

```java
// CustomerDTO.java
package com.example.dtos;

public class CustomerDTO {
    private int id;
    private String name;
    private String email;

    public int getId() {
        return id;
    }

    public void setId(int id) {
        this.id = id;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public String getEmail() {
        return email;
    }

    public void setEmail(String email) {
        this.email = email;
    }
}

// Mapper usage
RecordMapper<Customer, CustomerDTO> mapper = RecordMapperBuilder.from(Customer.class, CustomerDTO.class)
    .map(Customer::getId).to(CustomerDTO::setId)
    .map(Customer::getName).to(CustomerDTO::setName)
    .map(Customer::getEmail).to(CustomerDTO::setEmail)
    .build(tableManager);

CustomerDTO dto = mapper.map(customer);
```

## Working with Relationships

### Create Related Entities

```bash
# Create Order entity with relationship to Customer
norm4j:create-entity --name Order --package com.example.entities --tableName orders
```

Edit the Order entity to add the relationship:

```java
@Table(name = "orders")
@Join(
    columns = "customer_id", 
    reference = @Reference(table = Customer.class, columns = "id")
)
public class Order {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private int id;

    @Column(nullable = false)
    private String number;

    @Column(name = "customer_id")
    private int customerId;

    @Column(nullable = false)
    private BigDecimal total;

    // Getters and setters
}
```

### Generate Mapper with Relationships

```bash
# Generate mapper with order relationship
norm4j:generate-mapper \
    --entity Customer \
    --dto CustomerDTO \
    --mappings id:id,name:name,email:email \
    --joins orders:Order:OrderDTO
```

This generates a mapper that includes the orders:

```java
RecordMapper<Customer, CustomerDTO> mapper = RecordMapperBuilder.from(Customer.class, CustomerDTO.class)
    .map(Customer::getId).to(CustomerDTO::setId)
    .map(Customer::getName).to(CustomerDTO::setName)
    .map(Customer::getEmail).to(CustomerDTO::setEmail)
    .join(CustomerDTO::getOrders, Order.class, OrderDTO.class)
    .endJoin()
    .build(tableManager);

CustomerDTO customerDTO = mapper.map(customer);
List<OrderDTO> orders = customerDTO.getOrders();
```

## Query Building

### Basic Queries

```bash
# Explain a simple query
norm4j:explain-query --query "SELECT c FROM Customer c WHERE c.active = true"
```

This shows:
```
Original Query: SELECT c FROM Customer c WHERE c.active = true
Generated SQL: SELECT * FROM customer WHERE active = ?
Parameters: [true]
Explanation: This query selects all columns from the 'customer' table where active equals the parameter value.
```

### Complex Queries

```bash
# Explain a join query
norm4j:explain-query --query "SELECT c FROM Customer c JOIN c.orders o WHERE c.active = true AND o.total > 100"
```

This shows:
```
Original Query: SELECT c FROM Customer c JOIN c.orders o WHERE c.active = true AND o.total > 100
Generated SQL: SELECT c.* FROM customer c INNER JOIN orders o ON c.id = o.customer_id WHERE c.active = ? AND o.total > ?
Parameters: [true, 100]
Explanation: This query selects customer records with their orders where the customer is active and the order total exceeds 100.
```

## Schema Management

### Create Multiple Versions

```bash
# Create initial version
norm4j:create-migration --version v0.1 --description "Initial schema"

# Create second version
norm4j:create-migration --version v0.2 --description "Add email field"

# Create third version
norm4j:create-migration --version v0.3 --description "Add index on email"
```

This creates a complete migration chain:

```java
new SchemaSynchronizer(tableManager)
    .version("v0.1")
        .description("Initial schema")
        .schema()
            .enableAutoCreation(true)
        .endSchema()
    .endVersion()
    
    .version("v0.2")
        .description("Add email field")
        .execute("ALTER TABLE customer ADD COLUMN email VARCHAR(255)")
    .endVersion()
    
    .version("v0.3")
        .description("Add index on email")
        .execute("CREATE INDEX idx_customer_email ON customer(email)")
    .endVersion()
    
    .apply();
```

## Common Workflows

### Creating a New Entity with All Dependencies

```bash
# 1. Create the entity
norm4j:create-entity -n Product -p com.example.entities -t products

# 2. Validate the entity
norm4j:validate-entity -f src/main/java/com/example/entities/Product.java

# 3. Generate DDL
norm4j:generate-ddl -p com.example.entities -d postgresql -o ddl.sql

# 4. Create migration
norm4j:create-migration -v v0.1 -d "Add product table"

# 5. Generate DTO
norm4j:generate-dto -e Product -p com.example.dtos -f id,name,price,description

# 6. Generate mapper
norm4j:generate-mapper -e Product -d ProductDTO -m "id:id,name:name,price:price,description:description"
```

### Adding a Relationship Between Entities

```bash
# 1. Add @Join annotation to the entity
# (Edit the Java file manually)

# 2. Generate updated DDL
norm4j:generate-ddl -p com.example.entities -d postgresql

# 3. Create migration for the relationship
norm4j:create-migration -v v0.2 -d "Add relationship between Customer and Order"

# 4. Update the mapper to include the relationship
norm4j:generate-mapper -e Customer -d CustomerDTO -m "id:id,name:name,email:email" -j "orders:Order:OrderDTO"
```

### Creating a Migration for Schema Changes

```bash
# 1. Make changes to your entity (add/remove fields)
# (Edit the Java file manually)

# 2. Generate new DDL to see the changes
norm4j:generate-ddl -p com.example.entities -d postgresql

# 3. Create migration for the changes
norm4j:create-migration -v v0.3 -d "Add phone number field to customer"

# 4. Edit the migration to include the specific SQL changes
# (Add: .execute("ALTER TABLE customer ADD COLUMN phone VARCHAR(20)"))

# 5. Apply the migration
```

## Validation and Troubleshooting

### Validate Entities

```bash
# Validate a single entity
norm4j:validate-entity -f src/main/java/com/example/Customer.java

# Check entity information
norm4j:extract-entity-info -f src/main/java/com/example/Customer.java
```

### Validate Queries

```bash
# Validate query syntax
norm4j:validate-query -q "SELECT c FROM Customer c WHERE c.id = ?"

# Explain query to understand what SQL will be generated
norm4j:explain-query -q "SELECT c FROM Customer c WHERE c.active = true"
```

### Debugging Common Issues

**Issue**: "No metadata found for class"
```bash
# Solution: Register the class with MetadataManager
norm4j:validate-entity -f YourEntity.java
# Check that @Table annotation is present
```

**Issue**: "No primary key found"
```bash
# Solution: Add @Id annotation
norm4j:create-entity -n YourEntity -p your.package
# This will generate proper entity with @Id
```

**Issue**: "Join not found"
```bash
# Solution: Check @Join annotation
norm4j:validate-entity -f YourEntity.java
# Ensure @Join is properly configured
```

**Issue**: SQL syntax errors
```bash
# Solution: Check dialect compatibility
norm4j:generate-ddl -p your.package -d your-dialect
# Verify the generated SQL
```

## Integration with Build Tools

### Maven Integration

```bash
# Add norm4j Maven plugin to pom.xml

    -p com.example.entities \
    -d postgresql,mariadb,sqlserver \
    -v v0.1 \
    -o plugin-config.xml

# Then add to your pom.xml:
<build>
    <plugins>
        <!-- Add the generated plugin configuration here -->
    </plugins>
</build>
```

### Gradle Integration

While norm4j primarily provides Maven integration, you can still use it with Gradle:

```groovy
// Add dependencies
dependencies {
    implementation 'org.norm4j:norm4j-core:1.1.25'
    implementation 'org.norm4j:norm4j-postgresql:1.1.25'
    implementation 'org.postgresql:postgresql:42.7.2'
}

// Create a task to generate DDL
task generateDDL(type: JavaExec) {
    mainClass = 'org.norm4j.tools.DDLGenerator'
    classpath = sourceSets.main.runtimeClasspath
    args = ['com.example.entities', 'postgresql', 'src/main/resources/db']
}
```

## Tips and Tricks

### Quick Entity Creation

```bash
# Create multiple entities quickly
for entity in Customer Product Order Category; do
    norm4j:create-entity -n $entity -p com.example.entities -t ${entity,,}
done
```

### Batch DDL Generation

```bash
# Generate DDL for multiple dialects
for dialect in postgresql mariadb sqlserver; do
    norm4j:generate-ddl -p com.example.entities -d $dialect -o ddl-$dialect.sql
done
```

### Migration Versioning Strategy

```bash
# Use semantic versioning for migrations
norm4j:create-migration -v v1.0.0 -d "Initial release"
norm4j:create-migration -v v1.0.1 -d "Bug fixes"
norm4j:create-migration -v v1.1.0 -d "New features"
norm4j:create-migration -v v2.0.0 -d "Breaking changes"
```

### Organizing DTOs

```bash
# Create DTOs in separate packages
norm4j:generate-dto -e Customer -p com.example.dtos.customers -f id,name,email
norm4j:generate-dto -e Order -p com.example.dtos.orders -f id,number,total,customerId
norm4j:generate-dto -e Product -p com.example.dtos.products -f id,name,price,description
```

## Summary

The norm4j skill provides comprehensive support for working with norm4j, including:

1. **Entity generation**: Quick creation of properly annotated entities
2. **Schema management**: DDL generation and migration creation
3. **DTO mapping**: Automatic generation of DTOs and mappers
4. **Query analysis**: Understanding what SQL will be generated
5. **Project setup**: Complete project structure and configuration
6. **Validation**: Checking entities and queries for issues
7. **Database configuration**: Dialect-specific connection settings
8. **Testing**: Test infrastructure generation

This skill significantly reduces the boilerplate code needed when working with norm4j and helps ensure best practices are followed.
